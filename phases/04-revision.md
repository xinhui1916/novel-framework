# Phase 4: Revision — 修订（可选仅在需要时启用）

写小说大量工作是改。这个阶段专门处理**写作完成后的修改**。

## 回溯修改（改已有章节）

**触发方式：** 任何时候说"我想改第 X 章"或"把之前那章调整一下"。

流程：
1. 加载目标章节 + 当前 bible + 相关 tracker
2. 你说明要改什么（压缩/重写/增加/删减）
3. 修改正文
4. 同步更新：
   - 章节注册表（字数/状态）
   - 情节线/伏笔追踪（如果内容变了）
   - timeline（如果时间线变了）
   - 后续章节的衔接标记（避免断裂感）

## 章节调序

**触发方式：** "把第 5 章和第 8 章换一下"或"删掉第 3 章"。

流程：
1. 调换/删除文件
2. 重新编号
3. 更新章节注册表
4. 检查情节线连续性（调序后伏笔顺序是否还对）
5. 如果不连续，提示你并建议修补方案

## 全文删减

**触发方式：** "太长了，删掉 20%"或"这一章拖了"。

策略：
- 标记可删减段落（重复描写、拖沓对话、信息过慢的章节）
- 你确认后执行
- 不影响情节线的完整性和伏笔回收

## 合并章节

**触发方式：** "这两章太短，合并"。

流程：
1. 合并两章内容
2. 去重（衔接段、重复交代）
3. 重新设置章节钩子和结尾
4. 更新注册表
