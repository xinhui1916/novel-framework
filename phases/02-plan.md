# Phase 2: Plan — 大纲规划

## 前置条件
- Init 阶段已完成
- `bible/story-bible.md` 和角色卡已确认

## 流程

### 1. 加载设定
读取：
- `bible/story-bible.md` — 核心设定（当前版本）
- `bible/characters/_index.md` — 角色列表

### 2. 大纲规划

**先选叙事结构框架（全部可选，不预设）：**
- 三幕结构（经典，适合大部分类型）
- 非线性/倒叙/多线并行（悬疑、文学性作品）
- 起承转合（短篇、中式叙事）
- 回归线结构（角色驱动型）
- 无特定结构，按情节自然推进

选哪个取决于项目和你的偏好，不预设默认值。

**然后填充内容：**
- 拆解章节（至少到卷/部级别，推荐到章级别）
- 每章标注：POV、核心事件、字数目标
- 标记**钩子点**（每章末尾的悬念/爽点）
- 规划**伏笔埋设点**（提前标记哪些伏笔在哪章埋、哪章收）

如果拿不准结构 → 调 `writing-plans` skill 辅助。

### 3. 填充追踪文件
- `plot/outline.md` — 章节级大纲
- `plot/plot-threads.md` — 情节线 + 伏笔（使用模板）
- `plot/timeline.md` — 完善时间线
- `chapters/_index.md` — 章节注册表，每章一条记录

### 4. 用户审查
展示完整大纲，确认后进 Write 阶段。但注意：

> **大纲是当前路线图，随时可以重画。** 写到第 5 章发现第 2 章的伏笔换个方向更好 → 改大纲，不用重走 Plan 流程。

### 直接跳过 Plan 开始写
如果用户已经有完整构思，直接说"开始写"即可跳过 Plan。
灵感来了就写，大纲是辅助不是前提。

## 章节规划原则
- 每章 3000-5000 字（网文）或 2000-4000 字（实体书）
- 前三章必须有抓人的钩子
- 每章至少推进一条情节线
- 避免超过 3 条 POV 线并行（除非用户特意要求）
