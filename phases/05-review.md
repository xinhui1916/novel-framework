# Phase 4: Review — 审校（可选）

> **审校不是必须的。** 灵感在线时直接说"跳过审查"继续写。
> 如果感觉哪里不对劲、或者完稿后想统一打磨，再走这个流程。

## 什么时候需要审校

| 场景 | 建议 |
|------|------|
| 写得顺，状态好 | 跳过，继续写 |
| 感觉人设有点飘 | 做一次角色一致性检查 |
| 完稿了准备导出 | 做一次完整审查 + 去 AI 味 |
| 写到一半接不上前文 | 做一次批量检查，看看断层在哪 |

## 检查清单

### 一致性检查
- [ ] **角色一致性** — 角色名字、外貌特征、性格行为是否跨章一致
- [ ] **世界观一致性** — 设定规则是否被打破（如力量体系等级、科技水平）
- [ ] **时间线一致性** — 日期、季节、时间流逝是否合理
- [ ] **伏笔回收** — plot-threads.md 中标记的伏笔是否已收/待收
- [ ] **情节线完整性** — 每条 plot thread 有始有终

### 去 AI 味检查（6 道门禁）
- A. 禁用词替换（突兀的成语、套路化表达）
- B. 句式去套路（"他/她/它"开头的句子比例过高）
- C. 心理描写外化（"感到/觉得/意识到"替换为行动）
- D. 节奏打碎（排比句、长句堆砌打断）
- E. 对话去腔调（每个角色的说话方式是否区分度够）
- F. 结尾去升华（不要强行上价值/总结人生道理）

### 叙事检查
- [ ] 开头三章有足够钩子
- [ ] 每章有明确的情绪目标
- [ ] 场景转换不过于突兀
- [ ] 信息投放节奏合理（不堆设定、不谜语人）

## 输出
- 问题列表（分严重程度：阻断性/建议性/可选）
- 如果用户要求，直接输出润色版
