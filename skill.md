# 小说创作框架

用户想写小说时进入此框架。**核心原则：灵感优先于流程。框架是护栏，不是铁轨。**

## 第一步：项目评估（决定框架的介入程度）

进入框架后先聊几句，确定项目画像：

| 评估维度 | 选项 | 影响 |
|---------|------|------|
| 篇幅 | 短篇(＜1万) / 中篇(1-5万) / 长篇(5万+) / 网文连载(50万+) | 决定需要多少结构化追踪 |
| 写作风格 | 大纲党(先规划后写) / 即兴党(想到哪写到哪) / 混合 | 决定是否启用 Plan 阶段 |
| AI 味敏感度 | 在意 / 不在意 | 决定是否启用去 AI 味 6 道门禁 + 角色声音库 |
| 一致性要求 | 高(长篇复杂世界观) / 低(短篇日常) | 决定 bible 和跟踪的详细程度 |
| 字数管理 | 宽松(写好为止) / 固定目标(每章有字数要求) | 决定字数约束的严格度 |
| 叙事结构偏好 | 三幕 / 非线性 / 无特定偏好 | 决定 Plan 使用什么结构模板 |

根据评估结果，框架**只启用需要的功能**。例如：

- **短篇即兴** → 简单的 bible + 直接写 + 跳过审查
- **长篇玄幻** → 完整 bible + 大纲 + 伏笔追踪 + 一致性审查 + 去 AI 味
- **中篇混合** → bible + 简易大纲 + 按需审查

评估只用几句话交流，不搞填表。用户也可以随时说"这个算了"关闭某个功能。

## 创作循环（不是流水线）

```
                  ┌──────────────────────┐
                  │   写作（发现新想法）     │
                  └──┬───────────────────┘
                     │ "这个设定更好"
                     ▼
                  ┌──────────────────────┐
                  │   更新设定（bible 活的）│
                  └──┬───────────────────┘
                     │ "那前面要改一下"
                     ▼
                  ┌──────────────────────┐
                  │   回头调整              │
                  └──┬───────────────────┘
                     │ "好，继续"
                     ▼
                  ┌──────────────────────┐
                  │   写下一章              │
                  └──────────┬───────────┘
                             │
                 回到写作，循环
```

框架提供的节点（仅启用评估后确认的）：

| 节点 | 用途 | 可选/必选 |
|------|------|----------|
| Init | 设定起点（bible + 角色卡 + 角色声音库） | 必选一次，后续可随时更新 |
| Plan | 出大纲 + 伏笔安排 | **仅大纲党启用** |
| Write | 写章节 + 前情提要 + 角色声音参考 | 核心循环 |
| Revision | 回溯修改 + 调序 + 删减 + 合并 | **需要时再说"改一下第 X 章"** |
| Review | 审查一致性 + 去 AI 味 | **仅高一致性 / 在意 AI 味时启用** |
| Export | 导出 docx/pdf | 完稿时用 |

## 铁律（只有 2 条，不分项目类型）

1. **灵感与流程冲突时，灵感优先。** 写着写着发现更好的方向 → 先写下来，写完后去更新设定。
2. **bible 是活的。** 什么时候发现设定对不上正文了，什么时候改。改完标记变更记录。

## 项目目录结构（根据画像自动选）

**极简版**（短篇/即兴，省到极致）：
```
project-name/
├── bible.md               ← 一两句话的核心设定
├── chapter-01.md
└── progress.md            ← 进度
```

**完整版**（长篇/世界观复杂）：
```
project-name/
├── bible/
│   ├── story-bible.md        ← 核心设定（可更新，标注变更）
│   ├── characters/
│   │   ├── _index.md          ← 角色注册表
│   │   └── char-name.md       ← 角色详细设定
│   └── worldbuilding/
│       ├── _index.md          ← 世界观注册表
│       └── location-name.md   ← 地点/势力设定
├── plot/
│   ├── outline.md             ← 当前大纲
│   ├── plot-threads.md        ← 伏笔追踪
│   └── timeline.md            ← 时间线
├── chapters/
│   ├── _index.md              ← 章节注册表
│   └── ch-01-title.md
├── progress.md
└── style-sample.md            ← 文风样本（可选）
```

中间层级根据需要裁切。

## 引用规范
- `[REF:path.to.section]` — 常规引用指向 bible / plot 中的内容
- 知识库不可捏造，缺失时报"绑定失败"

## 字数管理（根据项目画像）
- **宽松模式**：写好为止，字数自然产生，不约束
- **目标模式**：有字数目标但灵活，如果剧情需要，2000 字也可以是一章
- **硬约束模式**：网文连载需要稳定字数，达不到标记未完成

## 断点恢复
- `progress.md` 记录当前进度，随时停随时续
- 审查暂停点按项目画像配置：长篇每 3-5 章一次，短篇不暂停
